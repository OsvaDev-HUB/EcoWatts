2. VULNERABILIDADES CRÍTICAS

2.1 Exposición de Secretos en .env

Ubicación: .env

Descripción: El archivo .env contiene credenciales sensibles en texto plano sin protección.

Evidencia:

SECRET_KEY=94b8e2d4f21a719c6e3b5a8f01cd3a2e5b9f8d7c6e4a1b2c3d4e5f6a7b8c9d0e
DATABASE_URL=postgresql://ecowatts_user:EcoWatts_SecurePass2025!@localhost:5432/Eco_Watts

Riesgo: Si este archivo se commitea al repositorio, todas las credenciales quedan expuestas públicamente.

Remediación:

  • Eliminar .env del repositorio (agregar a .gitignore)
  • Usar variables de entorno del sistema o servicio de secrets
  • Rotar todas las credenciales expuestas
  • Nunca committear archivos con secrets

2.2 Secret Key Débil por Defecto

Ubicación: app/config.py:7

Código vulnerable: SECRET_KEY = os.getenv('SECRET_KEY', 'dev_key')

Riesgo: Si SECRET_KEY no está configurada, usa "dev_key" como fallback. Sesiones pueden ser fácilmente comprometidas.

Remediación:

  • Eliminar el fallback por defecto
  • Validar que SECRET_KEY esté configurada al iniciar
  • Usar claves de al menos 256 bits (32 caracteres hex)

2.3 Sin Validación de Fortaleza de Contraseña

Ubicación: app/routes/auth.py:15-22

Descripción: No existe validación de longitud mínima, complejidad ni patrones en el backend.

Remediación:

  • Validar mínimo 8 caracteres
  • Requerir al menos una mayúscula, un número y un símbolo
  • Verificar que no sea una contraseña común
  • Mostrar feedback visual al usuario

3. VULNERABILIDADES DE ALTA SEVERIDAD

3.1 Sin Protección contra Fuerza Bruta

Ubicación: app/routes/auth.py:32-48

Descripción: No existe límite de intentos de login, CAPTCHA ni rate limiting.

Remediación:

  • Implementar flask-limiter o similar
  • Bloquear IP después de 5 intentos fallidos por 15 minutos
  • Agregar CAPTCHA después de 3 intentos
  • Registrar intentos fallidos para auditoría

3.2 Cookies sin Flags de Seguridad

Ubicación: app/__init__.py

Descripción: Las cookies de sesión no tienen HttpOnly, Secure ni SameSite configurados.

Remediación:

app.config.update(
    SESSION_COOKIE_SECURE=True,
    SESSION_COOKIE_HTTPONLY=True,
    SESSION_COOKIE_SAMESITE="Lax"
)

3.3 Headers de Seguridad Ausentes

Ubicación: app/__init__.py

Descripción: No se configuran HSTS, X-Frame-Options, X-Content-Type-Options, CSP.

Remediación:

  • Usar Flask-Talisman
  • Configurar Content-Security-Policy
  • HSTS con includeSubDomains
  • X-Frame-Options: DENY
  • X-Content-Type-Options: nosniff

3.4 Endpoint de Datos de Ejemplo Expuesto

Ubicación: app/routes/api.py:183-195

Descripción: /api/cargar-ejemplo permite agregar datos de ejemplo sin validación.

Remediación:

  • Eliminar endpoint en producción
  • O agregar flag de configuración DEBUG=True para habilitar

4. VULNERABILIDADES DE MEDIA SEVERIDAD

4.1 CSRF Token Condicional

Ubicación: app/templates/base.html:17-19

Descripción: El token CSRF se renderiza condicionalmente, causando errores si es None.

4.2 Exposición de Errores Detallados

Ubicación: app/routes/api.py:52-54

Descripción: Los mensajes de error exponen detalles del servidor.

4.3 Puerto de Base de Datos Expuesto

Ubicación: docker-compose.yml:13

Descripción: PostgreSQL expuesto al host (5433:5432) cuando debería ser solo interno.

4.4 Sin Rate Limiting en API

Ubicación: app/routes/api.py

Descripción: Endpoints API sin límite de requests, susceptibles a DoS.

4.5 Sin Sanitización XSS

Ubicación: app/routes/api.py:28-40

Descripción: Validación básica pero sin sanitización antes de guardar en BD.

4.6 Variables de Entorno en Docker

Ubicación: docker-compose.yml:21-23

Descripción: SECRET_KEY con fallback "dev_key" en variables de contenedor.

5. VULNERABILIDADES BAJAS

5.1 Dependencias sin Versionado Estricto

Ubicación: requirements.txt

Descripción: Las dependencias tienen versiones fijas pero sin rango mínimo de seguridad.

5.2 Debug Mode en Producción

Ubicación: config.py

Descripción: DEBUG no está explícitamente deshabilitado para producción.

7. RECOMENDACIONES GENERALES

  • Realizar pruebas de penetración antes del despliegue a producción
  • Implementar logs de auditoría para acciones sensibles
  • Usar HTTPS en producción con certificados válidos
  • Configurar backups automáticos de la base de datos
  • Documentar procedimientos de respuesta a incidentes
  • Realizar revisiones de código periódicas
  • Mantener actualizado el registro de dependencias
